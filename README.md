# 간판 글자 수집 게임 🪧

길거리 간판을 사진으로 찍어 **미션 단어의 음절을 한 글자씩 모으는** 모바일 웹 게임.

## 핵심 설계 (무한 로딩 해결)

원래 "무한 로딩"의 흔한 원인은 **백엔드 OCR API 키 누락 / CORS / 응답 형식 불일치**입니다.
이 앱은 그 원인을 **뿌리째 제거**하기 위해 **클라이언트 측 OCR(Tesseract.js, 한국어)** 를 씁니다.

- 외부 API 키 ❌ · 서버 ❌ · CORS ❌ → 그 계열 무한 로딩 원인이 애초에 없음
- 그래도 OCR은 실패할 수 있으므로 [`src/ocr.ts`](src/ocr.ts)에서:
  - 모든 호출을 **try/catch** + **타임아웃(15초)** 으로 감쌈 → 무한 대기 불가
  - 실패 시 **"글자를 인식하지 못했어요. 다시 찍어주세요"** + 재촬영 버튼
  - **개발용 에러 상세**(name/message/status)를 화면에서 펼쳐볼 수 있음
  - 큰 사진은 OCR 전 **다운스케일**(요청 막힘/지연 방지)
  - 앱 시작 시 워커를 **사전 로드**해 첫 인식 지연 방지

## 랜덤 단어

- 단어 풀은 [`src/words.ts`](src/words.ts) **한 곳**에서 관리 — 추가는 배열에만.
- 새로고침/«새 단어»마다 풀에서 **랜덤 선택**.
- 선택 단어를 **음절 단위로 자동 분리**해 빈 칸 생성 (예: `보물찾기 → [보][물][찾][기]`).
- 2~4음절이 섞여도 음절 수에 맞춰 칸이 자동 생성됨.

## 실행

```bash
npm install
npm run dev      # http://localhost:3000
```

> 카메라(`capture="environment"`)는 **모바일/HTTPS**에서 후면 카메라로 열립니다.
> 데스크탑에서는 파일 선택창이 뜨니, 간판 글자가 있는 사진을 골라 테스트하세요.

- [ ] OCR 실패 시에도 멈추지 않고 **재촬영**을 안내한다. (네트워크 끊고 테스트)
